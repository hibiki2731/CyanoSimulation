# 作品名 The Dungeon

## 1. 基本情報
* **制作者氏名**: 中島響
* **制作期間**: 2025年7月 〜 2026年5月
* **コンセプト**: 「成長による探索の進展」と「資源収集ループ」を融合させたゲーム体験
* **開発言語/API**: C++20 / DirectX 12

---

## 2. 動作環境と起動・終了方法

### 動作環境
* **OS**: Windows 10 / 11 (64bit)
* **その他**: DirectX 12対応グラフィックボード

### 起動方法
* DirectX12フォルダ内のDirectX12.exeを実行してください。

### 終了方法
* ゲームプレイ中にウィンドウ右上の `×` ボタンで終了します。

## 4. ゲームの概要と遊び方
* **目的**: 迷宮を探索して資源を集め、キャラクターを強化しながら深層を目指します。
* **特徴**: 集めた資源を使って装備を購入し、自身の能力を拡張することで、前回は進めなかったエリアへ到達できるようになります。

---

## 5. 技術的なアピールポイント
本プロジェクトは、3Dレンダリングやゲームエンジンの仕組みを低レイヤーから理解するため、DirectX12を用いて一から自作エンジンを構築しています。

* **安全で堅牢なメモリアーキテクチャの構築**
  * ゲームオブジェクトの管理において、スマートポインタ（`unique_ptr`）による所有権の管理と、生ポインタ配列による参照を完全に分離しました。
  * オブジェクト破棄時のデストラクタ内で、管理配列から自身の生ポインタを自動的に除去する「親子構造」の厳格なルールを設けることで、メモリリークやダングリングポインタを防ぐ安全な設計を実現しています。
* **専用パラメータエディタの自作によるワークフロー効率化**
  * 開発スピード向上のため、UIパラメータ等をリアルタイムに編集し、JSONファイルへ入出力できる独自エディタを作成しました。
  * オブジェクト名をキーとしたデータ整合性の担保などを実装し、コードの再コンパイルなしにレベルデザインを行える環境を構築しました。
* **各種APIの統合**
  * `XAudio2` と外部ライブラリを連携させたオーディオ再生基盤の構築。
  * `DWrite` を用いたテキストの生成。
  * 9スライス（9-slice）手法によるUIの柔軟なレンダリング。

## 6. ソースコードの見どころ
評価者の皆様には、特に以下のソースコードをご覧いただきたいです。

* **`Graphic.cpp`**
  * DirectX 12の初期化から、各種シェーダー（HLSL）の設定、描画コマンドの積込といったレンダリングパイプラインの構築を行っています。
* **`GUIDebugger.cpp` / `Object.cpp`**
  * エディタの処理と、エディタで設定した値の読み込みを行っています。
* **`Player.cpp` / `EnemyComponent.cpp`**
  * ダンジョンシーン中のプレイヤーとエネミーの処理を行っています。
